# Программа-методика испытаний

## 1. Общие сведения

### 1.1. Назначение документа

Настоящий документ содержит программу и методику испытаний программного комплекса для морфологического анализа текста на русском языке с разрешением омонимии и сохранением результатов в префиксном дереве, ключами которого являются последовательности слогов.

### 1.2. Цель испытаний

Целью испытаний является проверка соответствия разработанного программного комплекса требованиям, изложенным в техническом задании, а также проверка корректности работы всех модулей и системы в целом.

## 2. Объект испытаний

### 2.1. Наименование системы

Морфологический анализатор текста с разрешением омонимии и сохранением в префиксном дереве.

### 2.2. Состав системы

Система включает в себя следующие модули:
1. Модуль обработки текста (`text_processor.py`)
2. Модуль морфологического анализа (`morphology.py`)
3. Модуль разделения слов на слоги (`syllables.py`)
4. Модуль для работы с префиксным деревом (`trie.py`)
5. Модуль визуализации результатов (`visualization.py`)
6. Основной исполняемый модуль (`main.py`)

## 3. Условия проведения испытаний

### 3.1. Требования к программно-техническим средствам

- Операционная система: Windows 10/11, macOS 10.15+, Linux (Ubuntu 18.04+)
- Интерпретатор Python версии 3.7 и выше
- Установленные зависимости из файла `requirements.txt`
- Минимум 4 ГБ оперативной памяти
- Не менее 100 МБ свободного места на диске

### 3.2. Требования к персоналу

Персонал, проводящий испытания, должен:
- Знать основы Python
- Иметь представление о морфологическом анализе текста
- Уметь работать с командной строкой

## 4. Методика проведения испытаний

### 4.1. Тестирование модульных компонентов

#### 4.1.1. Тестирование модуля обработки текста

**Тестовые случаи:**
1. Чтение текстового файла
   - Входные данные: путь к существующему текстовому файлу
   - Ожидаемый результат: содержимое файла в виде строки
   
2. Нормализация текста
   - Входные данные: строка с лишними пробелами и переносами строк
   - Ожидаемый результат: строка с одиночными пробелами между словами
   
3. Токенизация текста с сохранением знаков препинания
   - Входные данные: текст с различными знаками препинания
   - Ожидаемый результат: список токенов, включая знаки препинания
   
4. Токенизация текста без сохранения знаков препинания
   - Входные данные: текст с различными знаками препинания
   - Ожидаемый результат: список токенов без знаков препинания
   
5. Очистка слова
   - Входные данные: слово с окружающими знаками препинания
   - Ожидаемый результат: слово без знаков препинания, приведенное к нижнему регистру

**Команда для запуска тестирования:**
```bash
python -m unittest tests/test_text_processor.py
```

#### 4.1.2. Тестирование модуля морфологического анализа

**Тестовые случаи:**
1. Анализ отдельного слова
   - Входные данные: отдельные слова разных частей речи
   - Ожидаемый результат: словари с морфологическими характеристиками
   
2. Извлечение морфологических тегов
   - Входные данные: слово и его морфологические характеристики
   - Ожидаемый результат: словарь с заполненными полями тегов
   
3. Разрешение омонимии
   - Входные данные: омонимичное слово и его контекст
   - Ожидаемый результат: корректное определение части речи

**Команда для запуска тестирования:**
```bash
python -m unittest tests/test_morphology.py
```

#### 4.1.3. Тестирование модуля разделения слов на слоги

**Тестовые случаи:**
1. Разделение на слоги стандартных слов
   - Входные данные: набор слов разной структуры
   - Ожидаемый результат: корректное деление на слоги
   
2. Обработка граничных случаев
   - Входные данные: пустая строка, односимвольное слово, слово без гласных
   - Ожидаемый результат: корректная обработка особых случаев
   
3. Подсчет количества слогов
   - Входные данные: различные слова
   - Ожидаемый результат: правильное количество слогов
   
4. Сбор статистики по слогам
   - Входные данные: список слов
   - Ожидаемый результат: корректный словарь статистики

**Команда для запуска тестирования:**
```bash
python -m unittest tests/test_syllables.py
```

#### 4.1.4. Тестирование модуля префиксного дерева

**Тестовые случаи:**
1. Создание узла дерева
   - Входные данные: слог для узла
   - Ожидаемый результат: узел с заданным слогом
   
2. Вставка слов в дерево
   - Входные данные: слово, разделенное на слоги, с данными морфологического анализа
   - Ожидаемый результат: слово добавлено в дерево
   
3. Поиск слова в дереве
   - Входные данные: слоги слова для поиска
   - Ожидаемый результат: данные слова или None, если слово не найдено
   
4. Поиск слов по префиксу
   - Входные данные: префикс (набор слогов)
   - Ожидаемый результат: список всех слов, начинающихся с данного префикса
   
5. Сбор статистики дерева
   - Входные данные: заполненное дерево
   - Ожидаемый результат: корректный словарь статистики
   
6. Сериализация и десериализация дерева
   - Входные данные: дерево с набором слов
   - Ожидаемый результат: восстановленное дерево идентично исходному

**Команда для запуска тестирования:**
```bash
python -m unittest tests/test_trie.py
```

### 4.2. Тестирование системы в целом

#### 4.2.1. Полный цикл обработки

**Тестовый случай:**
1. Обработка короткого тестового текста (1-2 абзаца)
   - Входные данные: небольшой текстовый файл
   - Ожидаемый результат: 
     - Успешное завершение программы
     - Созданы все выходные файлы
     - Отчет содержит разумные значения статистики

**Команда для запуска:**
```bash
python src/main.py --input tests/fixtures/short_text.txt --output-dir tests/output
```

#### 4.2.2. Обработка большого текста

**Тестовый случай:**
1. Обработка полного текстового файла (произведение А.П. Чехова "Воры")
   - Входные данные: файл "Исходный текст 1.txt"
   - Ожидаемый результат:
     - Успешное завершение программы без ошибок
     - Созданы все выходные файлы
     - Отчет содержит полную статистику анализа
     - Визуализации корректно демонстрируют результаты

**Команда для запуска:**
```bash
python src/main.py --input "Исходный текст 1.txt" --output-dir data/output
```

#### 4.2.3. Тестирование граничных условий

**Тестовые случаи:**
1. Обработка пустого файла
   - Входные данные: пустой текстовый файл
   - Ожидаемый результат: корректная обработка отсутствия текста
   
2. Обработка файла с нестандартными символами
   - Входные данные: текст с эмодзи, спецсимволами и др.
   - Ожидаемый результат: корректная обработка без ошибок
   
3. Ограничение количества слов
   - Входные данные: большой текст с ограничением в 100 слов
   - Ожидаемый результат: анализ только первых 100 слов

## 5. Протокол проведения испытаний

Результаты проведения испытаний документируются в отдельном документе "Протокол проведения испытаний" со следующей структурой:

1. Наименование испытания
2. Дата и время проведения
3. Исполнитель
4. Условия проведения
5. Ход испытания
6. Результаты (успех/неудача)
7. Комментарии
8. Выводы

## 6. Критерии успешного завершения испытаний

Испытания считаются успешно завершенными, если:

1. Все модульные тесты выполнены без ошибок
2. Система успешно обрабатывает как короткие, так и длинные тексты
3. Результаты анализа корректны и соответствуют заданным требованиям
4. Система корректно обрабатывает граничные случаи
5. Все выходные файлы создаются и содержат ожидаемые данные

## 7. Процедура устранения недостатков

В случае обнаружения недостатков в работе программы:

1. Недостаток фиксируется в протоколе
2. Определяется причина недостатка
3. Вносятся необходимые изменения в код
4. Проводится повторное тестирование модифицированного модуля
5. Проводится повторное интеграционное тестирование системы
